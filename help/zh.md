# 帮助 — Project Manager

## 概述
Project Manager 用于管理同一文件夹中的大量项目。
它扫描项目根文件夹，将一切显示在表格中，并可一键在任意项目中
启动 Claude Code CLI 或 Codex CLI。

主要功能：
• 带排序和搜索的所有项目表格
• 通过 DeepSeek 进行 AI 项目分析（项目是什么、处于什么状态）
• 在所选项目中启动 Claude / Codex
• 将多个项目作为 Windows Terminal 标签页打开（预设）
• 将重要项目置顶
• 带提醒的任务追踪器
• 多语言界面（5 种语言），即时切换

提示：随时按 F1 可打开此帮助。

## 项目列表
「项目」标签页显示项目根目录中的每个文件夹。

• 搜索 — 顶部的字段按名称、描述和技术栈进行筛选。
• 排序 — 点击列标题。
• 双击项目 — 启动 Claude Code。
• 右键单击 — 显示包含所有操作的上下文菜单。

「🔄 扫描（强制）」按钮会忽略缓存重新扫描文件夹。
「📂 文件夹」按钮在资源管理器中打开项目根目录。
「📁 数据」按钮打开存放设置和数据库的文件夹（%APPDATA%）。

## 置顶与排序
置顶项目始终显示在列表顶部，并以黄色高亮。

• 置顶 / 取消置顶 — 使用「📌 置顶」按钮或上下文菜单。
• 重新排序 — 用鼠标将置顶项目向上或向下拖动，或使用 Alt+↑ / Alt+↓。

顺序会自动保存，并在重启后保留。

## 启动 Claude 和 Codex
选择一个项目并按「▶ Claude」或「▶ Codex」— 会打开一个新的终端窗口，
代理已在项目文件夹中运行。

启动使用桌面上的脚本（Claude-BypassProxy、Codex-BypassProxy），
它们会配置环境并绕过代理。

「✨ 新建」会创建一个新的项目文件夹并在其中启动代理。

## 终端标签页与预设
「🖥 在标签页中打开」按钮会打开一个对话框，用于一次启动多个项目 —
每个项目位于各自的 Windows Terminal 标签页中，具有自己的颜色和标题。

工作流程：
1. 标记所需的项目（上下文菜单 →「标记到终端」）。
2. 打开对话框，按需重新排序并重命名标签页。
3. 按「🚀 打开」。

预设是已保存的项目集合。用「💾 另存为…」保存，下次只需一键即可
打开整个集合 — 适合早晨「打开我正在处理的所有项目」的例行操作。

标签标题可在列表中双击设置，或通过主表格的上下文菜单设置。
标题也会显示在主表格的「标签标题」列中。

## 恢复标题
执行 /resume 命令后，Claude 会重写终端标签标题。
「🏷 恢复标题」按钮可将标题恢复原状。

程序会查找打开的终端标签页，检测每个标签页中是哪个项目，
并将标签页重命名回其配置的标题。无需绑定预设 — 检测是动态的。

## 任务追踪器
「任务追踪」标签页是按项目分类的任务、想法和笔记列表。

• 任务可以关联到某个项目，也可以是「无项目」任务。
• 任务具有类型、状态、优先级、标签、截止日期和提醒。
• 提醒会在程序内触发；你也可以创建 Windows 系统提醒
  （通过任务计划程序），即使程序关闭也会触发。
• 「🚀 在 Claude / Codex 中测试」按钮会创建想法文件夹、写入 IDEA.md，
  并启动代理来推敲该想法。

## DeepSeek 分析
分析会描述项目是什么、如何运作以及处于哪个阶段。

• 「🤖 DS 分析」— 分析所选项目。
• 「🤖 DS：新项目」— 仅分析没有描述的项目。
• 「🤖 DS：全部」— 重新分析所有项目。
• 「⏹ 停止」— 中止批量分析。

结果会被缓存，并显示在右侧面板和表格中。

## 设置与语言
• 字体大小 — A− / A+ 按钮或 Ctrl + 鼠标滚轮。
• 语言 — 顶部的下拉框。界面即时切换，无需重启。
  提供俄语、英语、德语、西班牙语和中文。

所有设置和数据都存储在 %APPDATA%\ProjectManager 中。
每日备份保存在 Documents\ProjectManager-Backups。
